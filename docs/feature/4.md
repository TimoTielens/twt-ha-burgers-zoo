# Issue #4 — Chance-of-rain sensor

Branch: `feature/4_chance-of-rain-sensor`

## Problem

Expose the precipitation chance per forecast day as Home Assistant sensors.
Mirrors the temperature sensor (#3); reuses the #2 base entity and its
`_day_data` helper.

Requirements (from the ticket):
- One sensor per configured forecast day.
- Native unit `%`, `SensorStateClass.MEASUREMENT`.
- Value from `DayData.chance_of_rain`; `None` → unknown.

## Direction

**New `BurgersZooRainSensor(BurgersZooBaseEntity)`** in `sensor.py`:
- `key = "chance_of_rain"` → unique id `{entry_id}_chance_of_rain_{day}`.
- `_attr_native_unit_of_measurement = PERCENTAGE` (`"%"`).
- `_attr_state_class = SensorStateClass.MEASUREMENT`.
- **No device class** — Home Assistant has no sensor device class for
  precipitation probability, so this is a plain percentage sensor (unit + state
  class only). (Flag if you'd prefer to borrow `HUMIDITY` for the % icon; I'd
  rather leave it untyped than mislabel it as humidity.)
- `native_value` → `self._day_data.chance_of_rain` (guarded; `None` → unknown).
- Availability inherited from the base.
- English names: day 0 → "Chance of rain"; day k → "Chance of rain day +{k}".

**`async_setup_entry`** adds the rain sensor alongside the existing temperature
sensor, one of each per configured day:
```python
coordinator = entry.runtime_data
entities = []
for day in range(coordinator.forecast_days):
    entities.append(BurgersZooTemperatureSensor(coordinator, entry, day))
    entities.append(BurgersZooRainSensor(coordinator, entry, day))
async_add_entities(entities)
```

## Out of scope

- Other sensors (icon, business hours, suggestion) — #5–#7.
- Entity name translations (nl/de) — deferred polish.

## Testing

- **Unit:** a rain sensor for day 0 over `{0: chance_of_rain 10}` → `native_value == 10`,
  unit `%`, `state_class` measurement, `device_class` is `None`, `unique_id` ends
  `_chance_of_rain_0`.
- **Unit:** missing day → `native_value` None and unavailable.
- **Integration (real HA core + `mock_api`):** after setup (forecast_days=3), the
  entity registry has 3 rain entities for the config entry, and today's reads
  state `"10"` with unit `"%"`.
